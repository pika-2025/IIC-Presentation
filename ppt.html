Why Gradient Descent?

At its heart, training a machine learning model is all about optimization.
We need an efficient way to tweak the model's parameters to minimize error and improve predictions.
Gradient Descent is the foundational algorithm that powers this process, guiding our model to optimal performance.

The Landscape of Loss

Imagine a vast, hilly landscape. This is our Loss Function. The height at any point represents the model's error.
Our goal is to find the lowest valley in this landscape.
The Gradient is a vector that points in the direction of the steepest ascent.
To find the minimum, we take steps in the opposite direction of the gradient.

The First Step: Vanilla GD

\( \theta_{new} = \theta_{old} - \eta \nabla L(\theta) \)

Intuition: Take small, consistent steps downhill.

\( \theta \): Model's parameters (weights)
\( \eta \): Learning rate (controls step size)
\( \nabla L(\theta) \): Gradient of the loss function

Challenge: Can be very slow on flat areas of the loss landscape.

One Size Doesn't Fit All

Batch GD

Calculates the gradient using the entire dataset for a single update.

Accurate but extremely slow and memory-intensive for large datasets.

Stochastic GD (SGD)

Updates parameters using just one data point at a time.

Fast but noisy. The path to minimum is erratic and can oscillate.

Mini-Batch GD

A compromise: updates parameters using a small batch of data.

The standard choice. Balances accuracy and computational efficiency.

Gaining Momentum

Momentum:

Imagine a ball rolling down a hill. It picks up speed and doesn't get stuck in small bumps. This method adds a fraction of the previous update to the current one, helping to accelerate in the right direction.

Nesterov (NAG):

A smarter momentum. It "looks ahead" by calculating the gradient at a future approximate position, allowing it to slow down earlier and avoid overshooting the minimum.

Adaptive Learning Rates

Different learning rates for different parameters

AdaGrad

"Adaptive Gradient"

\( w_{t+1} = w_t - \frac{\eta}{\sqrt{v_t + \epsilon}} \nabla w_t \)
Gives frequent parameters a smaller learning rate and infrequent (sparse) parameters a larger one
Excellent for sparse data, like in NLP

Downside: The learning rate can decay too aggressively and become infinitesimally small, stopping learning altogether.

Fixing the Decay

RMSprop

Fixes AdaGrad's rapid decay by using an exponentially decaying average of squared gradients instead of summing them.

This prevents the denominator from growing endlessly, keeping learning going.

\( v_t = \beta v_{t-1} + (1-\beta)(\nabla w_t)^2 \)

AdaDelta

An extension of RMSprop that eliminates the need to set a base learning rate manually.

It uses a moving average of parameter updates in the numerator.

\( \Delta w_t = -\frac{\sqrt{u_{t-1}+\epsilon}}{\sqrt{v_t+\epsilon}}\nabla w_t \)

Adam: Best of Both Worlds

"Adaptive Moment Estimation" is often the default, go-to optimizer.

It combines the ideas of:

Momentum: Uses a moving average of the gradient (the "first moment")
RMSprop: Uses a moving average of the squared gradient (the "second moment")

This makes it efficient, robust, and well-suited for a wide range of problems.

Conclusion & Takeaways

We've journeyed from a simple downhill walk (Vanilla GD) to a sophisticated, adaptive algorithm (Adam).
Each new optimizer was developed to solve a specific problem: speed, noise, sparse data, or learning rate decay.
Choosing the right optimizer can significantly impact training speed and model performance.
